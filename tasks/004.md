# Task 004: BLS Price Seed & Crosswalk

## Context

This task creates the price data pipeline. It includes the BLS-to-USDA food crosswalk mapping and the seed script that fetches real prices from the BLS API and USDA category fallbacks. Task 003 (USDA seed) must be complete first because BLS prices are linked to specific food IDs that need to exist in the database.

## Files to Read

- PROJECT.md
- src/lib/db/index.ts
- src/lib/db/schema.ts
- scripts/seed-usda.ts

## Files to Write

- scripts/bls-crosswalk.ts
- scripts/seed-prices.ts
- tests/004_seed_prices.sh

## Steps

1. Create `scripts/bls-crosswalk.ts` — a manually curated mapping of ~60-80 BLS Average Price series IDs to USDA food descriptions:
   - Each entry: { blsSeriesId, blsItemName, usdaSearchTerm, unitConversionFactor, notes }
   - Include common foods: eggs, milk, bread, ground beef, chicken breast, rice, butter, cheese, apples, bananas, tomatoes, potatoes, etc.
   - Unit conversion factors convert BLS units (per lb, per gallon, per dozen) to price_per_100g
2. Create `scripts/seed-prices.ts` that:
   - **Step 1 — BLS item-level prices:** Read the crosswalk, look up each USDA food by search term in the foods table, fetch latest BLS prices via API v2 (`https://api.bls.gov/publicAPI/v2/timeseries/data/`), batch up to 50 series per request, convert to price_per_100g, insert into `price_data` with source='bls_average_price' and `bls_food_crosswalk` table
   - **Step 2 — USDA category fallback:** Insert category-level price estimates into `price_data` with source='usda_category_estimate', linked by food_group (no food_id)
3. Handle BLS API rate limits (500 queries/day, batch series per request)
4. Store the BLS period (e.g., "2026-M02") for freshness tracking
5. Use ON CONFLICT for idempotent re-runs
6. After all inserts, refresh the `food_computed` materialized view
7. Make runnable via `pnpm tsx scripts/seed-prices.ts`
8. Write test script to `tests/004_seed_prices.sh`

## Test Suggestions

- Crosswalk contains at least 50 BLS series mappings
- BLS API response is parsed correctly (extract series data, latest period)
- Unit conversion factors produce reasonable price_per_100g values
- BLS prices are linked to correct food_ids via crosswalk lookup
- Category fallback prices are inserted with food_group but no food_id
- Re-running doesn't create duplicates

## Definition of Done

Running `pnpm tsx scripts/seed-prices.ts` fetches BLS prices for ~60-80 common foods, inserts them with item-level food_id links, adds USDA category fallback prices for remaining food groups, and refreshes the materialized view. The crosswalk is well-documented and extensible.
