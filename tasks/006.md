# Task 006: Query Functions & Image Proxy

## Context

This task creates the server-side data access layer — the Drizzle query functions that power the table, and the Unsplash image proxy route. These are consumed by the table UI (Task 007). Depends on Task 002 (schema must exist). Can run in parallel with Tasks 003-005 (seed scripts), since query functions are defined against the schema, not the data.

## Steps

1. Create `src/lib/constants.ts` with:
   - Full nutrient name list with display names, units, and categories
   - Dietary preset definitions (Vegan, Keto, High-Protein, etc.) with default threshold values and filter rules
   - Food group list
2. Create `src/lib/queries/search-foods.ts` — a server action that queries the `food_computed` materialized view via Drizzle:
   - Parameters: query (text search), filters (dietary tags, nutrient ranges, food groups, additive exclusions), sortBy, sortDir, groupBy, page, pageSize
   - Uses `pg_trgm` similarity for fuzzy text search when query is provided
   - Builds dynamic WHERE clauses from filters
   - Supports ORDER BY any column, with GROUP BY for grouping
   - Returns paginated results with total count
3. Create `src/lib/queries/get-food-detail.ts` — a server action that returns the full nutrient profile for a single food:
   - Joins foods + all nutrients + price_data + food_images + food_tags
   - Organizes nutrients by category (macros, vitamins, minerals, amino acids, fatty acids)
4. Create `src/lib/queries/get-filter-options.ts` — a server action that returns distinct values for filter UI population:
   - Distinct food_groups, dietary tag values, allergen tag values, additive tag values
   - Min/max ranges for key nutrients (for range slider bounds)
5. Create the Unsplash image proxy route at `src/app/api/images/[foodId]/route.ts`:
   - Check `food_images` table first (cache hit → return cached URL)
   - On miss: call Unsplash API with food description, cache result in `food_images`, return URL
   - Include photographer attribution data
   - Return category placeholder on Unsplash failure or rate limit
   - Hide UNSPLASH_ACCESS_KEY from client
6. Write test script to `tests/006_queries.sh`

## Test Suggestions

- `searchFoods` returns paginated results with correct shape (id, name, food_group, nutrient values)
- Text search with `pg_trgm` returns fuzzy matches
- Filter by food_group correctly narrows results
- Sort by protein_per_100g desc returns highest-protein foods first
- `getFoodDetail` returns full nutrient breakdown organized by category
- `getFilterOptions` returns non-empty food_group list
- Image proxy route returns JSON with image_url and attribution
- Image proxy caches results in food_images table on first call
- Constants file exports valid dietary preset definitions

## Definition of Done

Three server action query functions are implemented and handle all combinations of search, filter, sort, group, and pagination. The Unsplash image proxy route works with cache-first strategy. Constants file defines all nutrient metadata and dietary presets. TypeScript compiles cleanly.
