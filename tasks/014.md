## Steps

1. Configure Vitest in `vitest.config.ts`:
   - TypeScript support, path aliases matching tsconfig
   - Separate unit and integration test configs
   - Setup files for msw handlers
2. Set up msw (Mock Service Worker) handlers for external APIs:
   - USDA FoodData Central API mock responses
   - BLS Average Price API mock responses
   - Unsplash API mock responses
3. Write unit tests:
   - `url-state.test.ts` — serialize/deserialize round-trip for all state fields, edge cases (empty values, special characters)
   - `csv-export.test.ts` — CSV generation, column filtering, special character escaping, empty data
   - `constants.test.ts` — dietary preset definitions produce valid filter configs, all presets have required fields
   - `price-conversion.test.ts` — BLS unit price → price_per_100g conversion math for various units (per lb, per dozen, per gallon)
4. Write integration tests (against local Postgres via Docker):
   - `seed-usda.test.ts` — USDA seed script parses API response, inserts valid rows
   - `seed-prices.test.ts` — BLS API parsing, crosswalk mapping, conversion factors
   - `seed-tags.test.ts` — tag derivation heuristics for known foods
   - `query-search.test.ts` — searchFoods returns correct results for various filter/sort/group combos
   - `query-detail.test.ts` — getFoodDetail returns full nutrient profile
   - `image-proxy.test.ts` — Unsplash proxy caches correctly, returns fallback on failure
5. Add test scripts to package.json: `test:unit`, `test:integration`, `test`
6. Perform accessibility review:
   - Ensure table has appropriate ARIA roles
   - All interactive elements are keyboard-navigable
   - Focus rings visible in all modes
   - Color is never the only indicator
7. Performance check:
   - Verify dynamic imports for filter panel, column picker, detail modal, tutorial
   - Check Lighthouse score targets (> 90)
   - Verify table filter/sort response time goal (< 500ms)
8. Write test script to `tests/014_testing.sh`

## Test Suggestions

- `pnpm test:unit` passes all unit tests
- `pnpm test:integration` passes all integration tests (requires Docker Postgres)
- msw correctly intercepts external API calls in tests
- URL state round-trips preserve all fields
- CSV export handles edge cases (empty data, special chars)
- Price conversion math is accurate for all BLS unit types
- Seed scripts produce expected data when run against mocked APIs
- Query functions return correct results for complex filter combinations
- Accessibility: tab through all interactive elements, ARIA labels present

## Definition of Done

Vitest is configured with unit and integration test suites. msw mocks external APIs. All unit tests pass (url-state, csv-export, constants, price-conversion). Integration tests cover seed scripts and query functions. Table has proper ARIA roles and keyboard navigation. Dynamic imports are in place for code splitting. The app builds and all tests pass.
