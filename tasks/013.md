# Task 013: Auth & Personalization

## Context

This task integrates Supabase Auth (auth-only, no Supabase DB) and builds the user personalization features. Authenticated users get saved preferences, dietary profiles, food exclusions, and accent color persistence. Depends on Task 002 (schema) since user data is stored in the self-hosted Postgres `user_preferences` and `saved_views` tables. Can run in parallel with Tasks 003-011.

## Steps

1. Create `src/lib/auth/client.ts`:
   - Initialize Supabase client for browser (auth-only, using NEXT_PUBLIC_SUPABASE_URL and NEXT_PUBLIC_SUPABASE_ANON_KEY)
   - Export auth helpers: signIn (email magic link), signInWithGoogle (OAuth), signOut, getSession, onAuthStateChange
2. Create `src/lib/auth/server.ts`:
   - Server-side Supabase client for session verification in server actions
   - Middleware helper: verify Supabase session, extract user_id
   - Wrapper function for protected server actions (verify session → run query scoped by user_id)
3. Update the Header component:
   - Anonymous: show "Sign In" button that opens a sign-in dialog (email input for magic link + "Sign in with Google" OAuth button)
   - Authenticated: show user avatar/email, dropdown with "Preferences" and "Sign Out"
4. Create server actions for user preferences:
   - `getUserPreferences(userId)` — fetch from user_preferences table
   - `updateUserPreferences(userId, prefs)` — upsert dietary_profile, excluded_foods, default_columns, accent_color
   - `getSavedViews(userId)` — list user's saved table configurations
   - `saveView(userId, name, configJson)` — save current table state as a named view with share_slug
5. Wire personalization into existing features:
   - On login: fetch user preferences, apply saved column selection, dietary profile filters, accent color
   - Accent color changes by authenticated users save to user_preferences (in addition to localStorage)
   - Dietary profile auto-applies as default filters (user can still override)
6. Write test script to `tests/013_auth.sh`

## Test Suggestions

- Supabase client initializes without errors
- Sign-in dialog renders email input and Google OAuth button
- Server action middleware correctly extracts user_id from Supabase session
- User preferences save and load from Postgres (upsert works)
- Accent color preference persists across sessions for authenticated users
- Dietary profile auto-applies as filters on login
- Unauthenticated users can still use all table features
- Saved views create unique share_slugs
- TypeScript compiles with no errors

## Definition of Done

Supabase Auth is integrated (email magic link + Google OAuth). Header shows sign-in/sign-out UI. Authenticated users can save and load preferences (columns, dietary profile, accent color, food exclusions). Preferences auto-apply on login. All data operations are scoped by user_id via server action middleware. Anonymous access remains fully functional.
