# Task 001: Project Scaffolding & Theme System

## Context

This is the first task — it creates the entire project foundation. Every subsequent task depends on this. The project is a Next.js 15 App Router application using pnpm, TypeScript strict mode, Tailwind CSS v4, and shadcn/ui.

## Steps

1. Initialize a Next.js 15 project with App Router, TypeScript strict mode, and pnpm
2. Install core dependencies: tailwindcss v4, shadcn/ui, drizzle-orm, drizzle-kit, @tanstack/react-table, @supabase/supabase-js, pg (node-postgres)
3. Configure Tailwind CSS v4 with CSS custom properties for accent colors (8-10 presets: blue, green, teal, orange, purple, rose, etc.) that work in both light and dark mode
4. Initialize shadcn/ui and add base components: button, dialog, popover, dropdown-menu, toggle, separator
5. Set up the theme system: ThemeProvider supporting light/dark/system mode detection, ThemeToggle component, AccentPicker component. Accent stored in CSS custom properties on :root, persisted in localStorage
6. Create the root layout with font setup (Inter or Geist Sans via next/font), theme provider wrapping, and global styles
7. Create the layout shell: Header (with theme toggle, accent picker, sign-in placeholder), Footer, MobileNav
8. Create a placeholder page.tsx with the header, a content area, and footer
9. Create .env.local.example with all required environment variables documented
10. Set up drizzle.config.ts pointing to the schema and migrations directories
11. Write test script to `tests/001_scaffolding.sh`

## Test Suggestions

- `pnpm build` completes without errors
- TypeScript compiles with no errors (`pnpm tsc --noEmit`)
- Dev server starts and serves the page
- Theme toggle switches between light and dark mode (check CSS class on html element)
- Accent color CSS custom properties are defined on :root

## Definition of Done

The project builds and runs in dev mode. The layout shell renders with a header (theme toggle, accent picker), footer, and a placeholder content area. Light/dark/system theme switching works. Accent color selection applies to CSS custom properties. All shadcn/ui base components are available for use.
