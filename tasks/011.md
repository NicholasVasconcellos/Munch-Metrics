# Task 011: Landing Page

## Context

This task builds the landing page sections that appear above the table on the homepage. The page is a single scroll — landing content on top, the table tool below. Depends only on Task 001 (project shell). Can run in parallel with Tasks 002-010 since it's purely presentational.

## Files to Read

- PROJECT.md
- src/app/page.tsx
- src/app/globals.css
- src/components/layout/header.tsx

## Files to Write

- src/components/landing/hero.tsx
- src/components/landing/how-it-works.tsx
- src/components/landing/stats-bar.tsx
- src/components/landing/scroll-transition.tsx
- src/app/page.tsx (update to include landing sections above table)
- tests/011_landing.sh

## Steps

1. Create `src/components/landing/hero.tsx`:
   - Bold headline (e.g., "Every nutrient. Every food. One table.")
   - 1-2 sentence subheadline explaining the value proposition
   - Primary CTA: "Explore Foods" — smooth-scrolls to the table section
   - Secondary CTA: "See How It Works" — opens the tutorial modal (wire to callback prop, actual modal in Task 012)
   - Visually striking design — subtle animated background or food-themed visual element
   - Responsive: stacks vertically on mobile
2. Create `src/components/landing/how-it-works.tsx`:
   - Three columns/cards with icons and short descriptions:
     - Search: "Find any food from 8,000+ whole foods and ingredients"
     - Filter: "Apply dietary restrictions, nutrient ranges, and custom metrics"
     - Discover: "Sort by protein-per-dollar, group by category, export your results"
   - Subtle micro-illustrations or icons (SVG, not stock photos)
   - Responsive: stack to single column on mobile
3. Create `src/components/landing/stats-bar.tsx`:
   - Horizontal strip with bold counter stats: "8,000+ Foods" | "40+ Nutrients" | "Monthly Price Data" | "100% Free"
   - Count-up animation triggered on scroll-into-view (Intersection Observer)
   - Styled with accent color
4. Create `src/components/landing/scroll-transition.tsx`:
   - Visual transition between landing content and the table section
   - Subtle gradient fade or decorative divider
   - Scroll-triggered animation to create a "dive into the data" feel
5. Update `src/app/page.tsx` to compose: Hero → How It Works → Stats Bar → Scroll Transition → Table Section (table rendered below)
6. Ensure landing sections use lazy loading / dynamic imports so they don't impact table performance
7. Write test script to `tests/011_landing.sh`

## Test Suggestions

- Hero section renders headline, subheadline, and both CTA buttons
- "Explore Foods" CTA smooth-scrolls to the table section
- How It Works shows three cards with icons
- Stats bar displays all four metrics
- Counter animation triggers on scroll into view
- Scroll transition renders between landing and table sections
- Layout is responsive (stacks correctly on mobile)
- `pnpm build` succeeds
- TypeScript compiles with no errors

## Definition of Done

The landing page renders above the table with a hero section (headline, CTAs), three-step "How It Works" explanation, animated stats bar, and a smooth scroll transition into the table. All sections are responsive and visually polished. The page scrolls naturally from landing content into the table tool.
