# Task 003: USDA Data Seed Script

## Context

This task creates the seed script that populates the database with ~8,000 foods from the USDA FoodData Central API. It imports Foundation Foods and SR Legacy datasets. Task 002 (schema) must be complete so the tables exist. The seed data is required before price data (Task 004) or tag derivation (Task 005) can run.

## Files to Read

- PROJECT.md
- src/lib/db/index.ts
- src/lib/db/schema.ts

## Files to Write

- scripts/seed-usda.ts
- tests/003_seed_usda.sh

## Steps

1. Create `scripts/seed-usda.ts` that connects to the database using the Drizzle client
2. Fetch USDA FoodData Central data via the FDC API (`/v1/foods/list` endpoint with `dataType` filter for Foundation and SR Legacy):
   - Paginate through results (200 items/page)
   - Implement rate limiting (stay under 1,000 requests/hour with delays between pages)
3. For each food item, insert into the `foods` table: map fdc_id, name (description), food_group (foodCategory), serving_size_g, serving_unit, description, data_source
4. For each food's nutrients array, bulk insert into the `nutrients` table: nutrient_name, nutrient_category (classify as macro/vitamin/mineral/amino_acid/fatty_acid/other), amount, unit, per_100g
5. Use Drizzle's batch insert for performance (batch 100-200 rows at a time)
6. Add progress logging (foods processed count, errors encountered)
7. Handle idempotency: use ON CONFLICT on fdc_id to upsert (so re-running is safe)
8. After all inserts, refresh the `food_computed` materialized view
9. Make the script runnable via `pnpm tsx scripts/seed-usda.ts`
10. Write test script to `tests/003_seed_usda.sh`

## Test Suggestions

- Script connects to the database without errors
- Script handles USDA API pagination correctly
- Foods are inserted with correct food_group mapping
- Nutrients are linked to correct food_id
- Nutrient categories are correctly classified (e.g., Protein → macro, Vitamin C → vitamin)
- Re-running the script doesn't create duplicates (upsert works)
- Progress output shows food count

## Definition of Done

Running `pnpm tsx scripts/seed-usda.ts` fetches USDA Foundation + SR Legacy foods, inserts them into the `foods` and `nutrients` tables, and refreshes the materialized view. The script is idempotent and rate-limited.
