# Task 002: Database Schema & Migrations

## Context

This task defines the entire Drizzle ORM schema and generates the initial migration. The schema covers all core tables (foods, nutrients, price_data, bls_food_crosswalk, food_images, food_tags), user tables (user_preferences, saved_views), the `food_computed` materialized view, and all indexes. Task 001 (scaffolding) must be complete first.

## Files to Read

- PROJECT.md
- drizzle.config.ts
- package.json

## Files to Write

- src/lib/db/index.ts
- src/lib/db/schema.ts
- src/lib/db/migrate.ts
- drizzle/migrations/ (generated by drizzle-kit)
- src/types/food.ts
- src/types/filters.ts
- src/types/table.ts
- tests/002_schema.sh

## Steps

1. Create the Drizzle client in `src/lib/db/index.ts` — node-postgres pool wrapping with drizzle(), configured from DATABASE_URL env var with SSL
2. Define all tables in `src/lib/db/schema.ts` using Drizzle's pgTable:
   - `foods` (id uuid PK, fdc_id integer unique, name text indexed, food_group text indexed, food_subgroup text, data_source enum, serving_size_g numeric, serving_unit text, description text, timestamps)
   - `nutrients` (id uuid PK, food_id FK, nutrient_name text, nutrient_category enum, amount numeric, unit text, per_100g numeric)
   - `price_data` (id uuid PK, food_id FK nullable, food_group text, bls_series_id text nullable, price_per_unit numeric, unit text, price_per_100g numeric, source enum, period text, region text, updated_at)
   - `bls_food_crosswalk` (id uuid PK, bls_series_id text unique, bls_item_name text, food_id FK, unit_conversion_factor numeric, notes text nullable)
   - `food_images` (id uuid PK, food_id FK unique, image_url text, thumbnail_url text, source enum, photographer_name text nullable, photographer_url text nullable, fetched_at)
   - `food_tags` (id uuid PK, food_id FK indexed, tag_type enum, tag_value text indexed)
   - `user_preferences` (id uuid PK, user_id text unique, dietary_profile jsonb, excluded_foods uuid[], default_columns text[], accent_color text nullable, updated_at)
   - `saved_views` (id uuid PK, user_id text nullable, name text, config_json jsonb, share_slug text unique indexed, created_at)
3. Export TypeScript types inferred from the schema for use throughout the app (food.ts, filters.ts, table.ts)
4. Create a custom SQL migration for the `food_computed` materialized view (the COALESCE join of foods + nutrients + prices + images) and all indexes (pg_trgm GIN index, composite indexes)
5. Run `drizzle-kit generate` to produce the migration files
6. Create the migration runner (`src/lib/db/migrate.ts`)
7. Write test script to `tests/002_schema.sh`

## Test Suggestions

- `drizzle-kit generate` succeeds and produces migration SQL
- TypeScript compiles with all schema types (`pnpm tsc --noEmit`)
- Migration SQL contains CREATE TABLE for all 8 tables
- Migration SQL contains CREATE INDEX statements including the pg_trgm GIN index
- Migration SQL contains the food_computed materialized view definition

## Definition of Done

All 8 tables and the materialized view are defined in Drizzle schema. Migration files are generated. TypeScript types are exported. The Drizzle client is configured and ready for use by seed scripts and query functions.
