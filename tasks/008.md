# Task 008: Filter System

## Context

This task builds the complete filtering system — the filter bar with active chips, the filter panel (drawer/sidebar), dietary presets with editable thresholds, nutrient range filters, food group multi-select, and additive exclusion filters. Depends on Task 007 (core table) since filters integrate with the table's data fetching and URL state. Can run in parallel with Tasks 009 and 010.

## Steps

1. Update `src/components/table/filter-bar.tsx` — renders active filters as removable chips/tags below the search bar. Each chip shows the filter name and value, with an X to remove. "Add Filter" button opens the filter panel.
2. Create `src/components/table/filter-panel.tsx` — sidebar drawer on desktop, bottom sheet on mobile. Contains all filter controls organized in sections. "Apply" and "Clear All" buttons.
3. Create `src/components/filters/dietary-filter.tsx`:
   - Dropdown of preset diet profiles (Vegan, Vegetarian, Pescatarian, Keto, Paleo, Gluten-Free, Dairy-Free, Nut-Free, Low-Sodium, High-Protein)
   - Selecting a preset auto-populates filter rules with default thresholds from constants
   - Each threshold is editable (slider or number input) after preset loads
   - Individual rules can be toggled on/off
   - Active preset shown as chip (e.g., "Keto (modified)" if thresholds changed)
4. Create `src/components/filters/nutrient-range.tsx` — min/max sliders or number inputs for any nutrient (e.g., "Protein > 20g per 100g", "Sodium < 500mg"). Fetches bounds from getFilterOptions.
5. Create `src/components/filters/additive-filter.tsx` — multi-select for common additives to exclude (BHA, BHT, MSG, artificial sweeteners, artificial colors)
6. Create `src/components/filters/tag-filter.tsx` — food group multi-select checkboxes populated from getFilterOptions
7. Wire all filters into the URL state system — each filter type gets serialized to URL params (diet, allergens, nutrient ranges, food groups, additive exclusions)
8. Ensure searchFoods query receives and applies all filter parameters
9. Write test script to `tests/008_filters.sh`

## Test Suggestions

- Filter bar renders active filter chips
- Removing a chip updates the URL and re-fetches data
- Filter panel opens as drawer on desktop
- Selecting "Keto" preset populates carb/fat threshold filters

- Nutrient range slider filters results correctly
- Food group multi-select narrows results
- All filter state round-trips through URL params
- "Clear All" removes all filters and resets URL
- TypeScript compiles with no errors

## Definition of Done

The filter system is fully functional: dietary presets with editable thresholds, nutrient range filters, food group multi-select, and additive exclusion. All filters appear as removable chips in the filter bar. Filter state is serialized to URL params. Filters correctly narrow the table results via the searchFoods query.
