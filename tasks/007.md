# Task 007: Core Data Table

## Context

This task builds the main table UI — the central feature of the app. It integrates TanStack Table v8 in server-side mode with the query functions from Task 006. Includes column definitions, sorting, pagination, search with autocomplete, column picker, and column resizing. The URL state hook is also created here since the table is the primary consumer.

## Steps

1. Create `src/lib/url-state.ts` — serialize/deserialize table state (query, sort, direction, columns, page, pageSize) to/from URL search params using short keys (q, sort, dir, cols, page, ps)
2. Create `src/hooks/use-url-state.ts` — React hook that syncs table state with URL params (read on mount, update URL on state change using Next.js router)
3. Create `src/hooks/use-debounce.ts` — generic debounce hook (used for search input)
4. Create `src/hooks/use-food-query.ts` — data fetching hook that calls the `searchFoods` server action when table state changes, manages loading/error states
5. Create `src/components/table/columns.tsx` — TanStack Table column definitions for all fields:
   - Default visible: Image (thumbnail), Name, Calories, Protein, Fat, Carbs, Food Group
   - All available: + Fiber, all vitamins/minerals, Serving Size, Avg Price/100g, Calories/Dollar, Protein/Dollar
   - Use monospace/tabular-nums for numeric columns
6. Create `src/components/table/sort-indicator.tsx` — arrow indicator on sorted column header (asc/desc/none cycle)
7. Create `src/components/table/column-picker.tsx` — popover with checkboxes for each available column, visibility state synced to URL
8. Create `src/components/table/data-table.tsx` — main TanStack Table wrapper:
   - Server-side mode (manualSorting, manualPagination)
   - Column resizing via drag
   - Search bar above table (full-width, debounced 300ms, with autocomplete dropdown from fuzzy search)
   - Column picker button in toolbar
   - Pagination controls below table (Previous/Next, page indicator, page size selector 25/50/100)
   - "Showing X-Y of Z results" indicator
   - Loading spinner overlay during data fetch
   - Skeleton shimmer on initial load
9. Create a placeholder `filter-bar.tsx` that renders active filter chips (actual filter logic in Task 008)
10. Update `src/app/page.tsx` to render the data table as the main content below the header
11. Write test script to `tests/007_data_table.sh`

## Test Suggestions

- Table renders with default columns (Image, Name, Calories, Protein, Fat, Carbs, Food Group)
- Clicking a column header toggles sort direction (asc → desc → none)
- URL updates when sort changes (e.g., ?sort=protein_per_100g&dir=desc)
- Search input filters results with debounce
- Pagination controls work (next/previous, page size change)
- Column picker toggles column visibility
- Column visibility persists in URL (?cols=name,calories,protein)
- Loading spinner appears during data fetch
- TypeScript compiles with no errors

## Definition of Done

The main data table renders with server-side data, supports single-column sorting with visual indicators, pagination with configurable page size, instant search with debounce, column visibility toggling via picker, and column resizing. All table state is synced to URL params. A spinner overlay shows during loading.
